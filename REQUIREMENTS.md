# 极简待办 (Lite-Todo) 需求分析文档

## 1. 产品概述 (Product Overview)

本项目旨在开发一款极致轻量化、纯本地的桌面端待办事项应用。
核心设计理念参考"敬业签"，追求简单、高效、无干扰。
**核心价值**：极速启动、纯净无广、数据私有、操作便捷。

## 2. 核心功能需求 (Functional Requirements)

### 2.1 基础架构

- **纯本地运行**：无需注册登录，无云端同步（V1版本），数据存储在本地。
- **数据库**：使用 `better-sqlite3` 存储所有分类和待办数据，保证读写性能。

### 2.2 窗口与交互

- **自定义标题栏**：
  - 去除原生系统标题栏。
  - **置顶功能**：核心需求。在最小化按钮左侧提供"置顶/取消置顶"切换按钮。
  - **窗口控制**：置顶、最小化、关闭。
- **视觉风格**：
  - 深色/暗黑主题（参考敬业签）。
  - 紧凑布局，最大化信息展示密度。

### 2.3 待办管理 (Todo Management)

- **快速新增**：顶部常驻输入框，支持回车快速添加。
- **任务列表**：
  - 展示待办内容。
  - 支持标记完成（Checkbox）。
  - 支持删除。
- **分类/便签夹**：
  - 左侧或顶部提供分类切换（如：工作、生活、备忘）。
  - 支持新建、重命名、删除分类。

### 2.4 数据字段定义

- `Task` 表：
  - `id`: 主键
  - `content`: 文本内容
  - `is_completed`: 完成状态 (0/1)
  - `category_id`: 所属分类ID
  - `created_at`: 创建时间
  - `order`: 排序权重

- `Category` 表：
  - `id`: 主键
  - `name`: 分类名称
  - `order`: 排序权重

## 3. 技术架构 (Technical Architecture)

### 3.1 技术栈

| 模块         | 选型               | 说明                |
| :----------- | :----------------- | :------------------ |
| **构建工具** | Electron + Vite    | 现代化的构建体验    |
| **前端框架** | Vue 3 + TypeScript | 组合式 API 开发     |
| **UI 样式**  | TailwindCSS + Sass | 高效原子化 CSS 开发 |
| **数据存储** | Better-SQLite3     | 高性能本地数据库    |
| **包管理**   | **pnpm**           | 硬性规定            |

### 3.2 目录结构规划 (初步)

```
/
├── electron/         # Electron 主进程代码
├── src/              # Vue 渲染进程代码
│   ├── components/   # UI 组件
│   ├── layout/       # 布局组件 (包含自定义标题栏)
│   ├── db/           # SQLite 封装层
│   └── styles/       # 全局样式
├── AGENT.md          # 开发准则
└── REQUIREMENTS.md   # 需求文档
```

## 4. 界面原型参考 (UI Reference)

_参考对象：敬业签_

- **布局**：
  - **Header**: 极窄，包含 Logo(可选)、置顶、最小化、关闭。
  - **Sub-Header**: 分类 Tab 或 当前分类标题 + 更多操作。
  - **Input Area**: 醒目的输入框，支持时间/日期快速选择（V2考虑，V1通过文本解析或简化）。
  - **Content Area**: 待办列表，紧凑排列。
  - **Footer** (可选): 简单的统计或工具栏。

## 5. 开发阶段规划 (Phases)

- **Phase 1: 脚手架与核心UI**
  - 搭建 Electron+Vite+Vue3 环境。
  - 实现无边框窗口及自定义控制按钮（含置顶）。
- **Phase 2: 数据库与CRUD**
  - 集成 SQLite。
  - 实现分类和任务的增删改查。
- **Phase 3: 交互优化**
  - 拖拽排序（预留）。
  - 快捷键支持（预留）。
