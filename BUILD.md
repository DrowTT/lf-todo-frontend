# Windows 打包说明

## 打包命令

项目已配置好 Windows 打包功能，使用以下命令即可打包：

```bash
# 完整打包（推荐）
pnpm run build:win

# 或者分步执行
pnpm run build        # 先构建项目
pnpm exec electron-builder --win  # 再打包
```

## 打包产物

打包完成后，安装包将生成在 `dist` 目录：

```
dist/
├── lf-todo-1.0.0-setup.exe       # NSIS 安装包
└── lf-todo-1.0.0-portable.exe    # 便携版
```

### 文件说明

**NSIS 安装包** (`lf-todo-1.0.0-setup.exe`)

- 完整的 Windows 安装程序
- 支持自定义安装路径
- 自动创建桌面和开始菜单快捷方式
- 支持卸载

**便携版** (`lf-todo-1.0.0-portable.exe`)

- 无需安装，双击即可运行
- 适合 U 盘携带使用
- 数据存储在用户目录

## 首次打包注意事项

1. **确保已安装依赖**

   ```bash
   pnpm install
   ```

2. **打包时间较长**
   - 首次打包需要下载 Electron 二进制文件
   - 预计耗时 5-15 分钟（取决于网络速度）
   - 后续打包会快很多

3. **磁盘空间**
   - 确保至少有 500MB 可用空间
   - 打包产物约 100-150MB

## 测试安装包

打包完成后，建议测试：

## 常见问题

### Q: 打包失败怎么办？

A: 检查以下几点：

- 确保已运行 `pnpm install`
- 确保 Node.js 版本 >= 18
- 删除 `node_modules` 和 `dist` 后重试
- 检查网络连接（首次打包需要下载文件）

### Q: 安装包太大怎么办？

A: 这是正常的，因为包含了完整的 Electron 运行时和 Node.js。可以考虑：

- 使用便携版（体积相同但无需安装）
- 使用 7z 压缩后分发

### Q: 如何更新版本号？

A: 修改 `package.json` 中的 `version` 字段，然后重新打包。
